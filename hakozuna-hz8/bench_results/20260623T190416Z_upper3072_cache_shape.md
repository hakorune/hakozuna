# Upper3072 Cache Shape Audit

Date: 2026-06-23

Goal:

```text
test a narrow size-policy candidate that preserves p2-v0 for <=2048
and only adds a 3072-byte class for 2049..3072
```

Builds:

```text
p2-v0:
  h8_bench_release

upper3072-v0:
  h8_bench_release_upper3072
  H8_CLASS_MAP_UPPER3072
```

Inputs:

```text
local:
  runs=5 threads=16 iters=100000 size=16..2048 remote=0 interleaved=0

interleaved:
  runs=5 threads=16 iters=100000 size=16..4096 remote=90 interleaved=1 live_window=4096

phase:
  runs=5 threads=16 iters=100000 size=16..4096 remote=90 interleaved=0
```

## Code Shape

```text
upper3072 preserves p2-v0 class selection for <=2048
3072 class uses factor-3 decode without div/idiv instructions
no __tls_get_addr observed in the objdump audit
```

Whole-file divide counts include unrelated cold code and are not treated as a
leaf verdict.

## Interpretation

```text
upper1p5:
  HOLD as default due guard local0 -27.2%

upper3072:
  CONDITIONAL GO as the best current size-policy candidate
  local regression is within noise-level for R5
  interleaved remote90 regression is small but needs paired R10x2
```

The rc1 default remains `p2-v0`.  The next promotion step is a paired R10 x 2
gate before any behavior cutover.
