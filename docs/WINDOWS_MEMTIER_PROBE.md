## Current Finding

`configure.ac` shows the upstream Unix build expects:

- `pthread`
- `zlib`
- `libevent`
- optional TLS via OpenSSL
- autotools / pkg-config for dependency discovery

The first Windows-native probe should therefore start from:

- [memtier_benchmark.cpp](/C:/git/hakozuna-win/private/bench-assets/windows/memcached/client/memtier_benchmark-2.2.1/memtier_benchmark.cpp)
- then optionally [client.cpp](/C:/git/hakozuna-win/private/bench-assets/windows/memcached/client/memtier_benchmark-2.2.1/client.cpp)

The expected early blockers are Unix headers and crash-handler APIs such as:

- `pthread.h`
- `unistd.h`
- `getopt.h`
- `execinfo.h`
- `ucontext.h`
- `sys/resource.h`
- `sys/utsname.h`

  - before the final fix, per-connection EOF tagging showed only `client 37..63` flapping, which matched the false capacity ceiling
  - Windows now uses `stats_state.curr_conns + stats_state.reserved_fds` for the fast reject path
  - verification reruns after the fix:
